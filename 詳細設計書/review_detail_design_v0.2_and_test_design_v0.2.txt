【新規指摘（再レビューで発見）】
================================================================================

────────────────────────────────────────────────────────────────────────────────
新規-1（🟡 要改善）｜NewsCard.jsxが削除したsource_urlカラムを参照したままになっている
対象：detail_design_v0.2.md セクション3-2（NewsCard.jsx）
────────────────────────────────────────────────────────────────────────────────

【問題点】
致命-2の対応としてDBから source_url カラムは削除されたが、
フロントエンドの NewsCard.jsx（セクション3-2）では以下の記述が残っている：

  <a href={article.source_url} target="_blank" rel="noopener noreferrer">

APIレスポンスで link を source_url として返す設計になっているため、
フロントエンド側のコードは動作するが、設計書上でAPIレスポンスのフィールド名が
source_url であることを明示していないと混乱の原因になる。

【変更案】
APIレスポンス生成のコード例（セクション1-7またはAPIエンドポイント設計箇所）に
以下を明示する：

  # GET /api/articles レスポンス生成時
  {
      ...
      "source_url": article.link,   # linkカラムをsource_urlとして返す
      ...
  }


────────────────────────────────────────────────────────────────────────────────
新規-2（🟢 軽微）｜title_jaのDDLコメントが変更履歴記載と実際の内容で食い違う
対象：detail_design_v0.2.md セクション2-1（articlesテーブルDDL）
────────────────────────────────────────────────────────────────────────────────

【問題点】
変更履歴v0.2に「title_jaのNULL設計意図をDDLコメントで明記（軽微-1）」とあるが、
セクション2-1のDDL本文では title_ja TEXT, のままでコメントが追加されていない。

【変更案】
DDLに以下のコメントを追記する：

  title_ja    TEXT,   -- 翻訳済み見出し（エラー時はtitle_originalと同値・NULLにならない設計）

[NEW-YEL-01] 構成図のSQLite DBテーブル一覧にgame_scheduleが未反映
該当箇所：§1-2 構成図

現状：
  │  articles / fetch_logs / app_settings / users

§4-2で新規追加されたgame_scheduleテーブルが構成図のDB部分に含まれていない。
v0.1から引き継がれたままの記述で、v0.3時点の設計と乖離している。
構成図は設計の全体感を伝える最重要成果物であり、テーブル構成との不一致は
読者の混乱を招く。

修正案：
  │  articles / fetch_logs / app_settings / users / game_schedule

[NEW-GRN-01] D-03の残課題表記と§3-1の記述が矛盾
該当箇所：§8 残課題D-03 / §3-1 C-05バッチフロー

  §8 D-03 ：「リアルタイム更新は不要と判断しているが詳細設計で確定」
  §3-1 注記：「リアルタイム更新は行わない（D-03として許容済み）」

§3-1では許容済みと書いており実質確定済みとして扱っているが、§8では未決として残している。
どちらかに統一すること。

修正案（どちらか選択）：
- 方針確定ならD-03を§8の「確定済み」リストに移し§3-1を維持
- 未確定なら§3-1の「D-03として許容済み」を「D-03として詳細設計で確定」に変更

--------------------------------------------------------------------------------

[NEW-GRN-02] Dedup Processorでis_duplicate=Trueの場合のフロー終端が不明示
該当箇所：§3-1 バッチ処理フロー

現状：
  │ 類似度80%以上 → is_duplicate=True でDB保存（非表示・修正可能）
  │ 類似度80%未満 → 処理継続
  ▼
  [Spursフィルタ]

is_duplicate=Trueの場合にDB保存後に処理が止まる（Spursフィルタ・Claude Clientを
経由しない）ことがフロー図から読み取りにくい。

修正案（DB保存の行に追記）：
  │ 類似度80%以上 → is_duplicate=True でDB保存（非表示・修正可能）→ 以降の処理をスキップ

## No.1 — Claude API 月次上限の確認・設定

### 目的
Claude API の過剰課金を防ぐため、月次上限 $20 を設定する。

### 手順

1. ブラウザで以下にアクセス:  
   `https://console.anthropic.com`

2. 対象の **Workspace** を選択する

3. 左サイドバー → **Settings** → **Limits** を開く

4. **Monthly Spend Limit** の現在値を確認する

5. 値が `$20` でない場合（または未設定の場合）:
   - 入力フィールドに `20` を入力
   - **Save** ボタンをクリックして保存

### 確認基準
- Monthly Spend Limit = **$20** が表示されていること

### 備考
- 上限超過後は Claude API 呼び出しがエラーになる（アプリは翻訳不能になるが課金は止まる）
- 月次試算 $0.93 に対して **21倍のマージン**（risk_report §2-3参照）

## No.2 — Claude API 最新料金の確認

### 目的
設計時の試算（$0.93/月）が現行料金と大きく乖離していないかを確認する。

### 手順

1. ブラウザで以下にアクセス:  
   `https://www.anthropic.com/pricing`

2. モデル **`claude-haiku-4-5-20251001`** の料金を確認する

3. 以下の試算と照合する:

   | 項目 | 値 |
   |------|---|
   | 使用モデル | claude-haiku-4-5-20251001 |
   | Input トークン | 4,000 tokens × 4記事 × 730回/年 ÷ 12 = 約970K tokens/月 |
   | Output トークン | 2,000 tokens × 4記事 × 730回/年 ÷ 12 = 約485K tokens/月 |
   | **現行レート（Input）** | **$1.00 / 1M tokens**（2026-06-14 anthropic.com/pricing 実測値） |
   | **現行レート（Output）** | **$5.00 / 1M tokens**（2026-06-14 anthropic.com/pricing 実測値） |
   | 月額試算内訳 | Input: 970K × $1.00 = **$0.97** / Output: 485K × $5.00 = **$2.43** |
   | **月額試算合計** | **$3.40/月** |

   > ⚠️ **要注意：設計書（requirements_p2_v0.12.md）の試算額 $0.93/月 は旧料金ベース。**  
   > 現行レートでは **$3.40/月** となり、プロジェクト月額合計も **$10.65/月**（Render $7.25 + Claude $3.40）に修正が必要。  
   > ただし Claude API 上限 $20 に対して **5.9倍のマージン**があるため、コストリスクは依然として低い。

4. 現行レートで以下の許容範囲を確認する:

   | 区分 | Input（/1M tokens） | Output（/1M tokens） | 月額試算 | 判断 |
   |------|--------------------|--------------------|---------|------|
   | 現行レート | $1.00 | $5.00 | $3.40 | ✅ 問題なし |
   | **許容上限（要再検討の閾値）** | **$2.00** | **$10.00** | **$6.79** | ⚠️ この水準以下なら許容 |
   | 超過（要再検討） | $2.00超 | $10.00超 | $6.79超 | ❌ 要再評価 |

   > 許容上限の根拠: 現行レートの **2倍** 以内（月額 $6.79 ≤ Claude API 上限 $20 の 1/3）

5. 現行料金が上記許容上限を超える場合は requirements_p2_v0.12.md を参照して使用量・モデル選択を再評価する

### 確認基準
- Input レートが **$2.00/1M tokens 以下** であること
- Output レートが **$10.00/1M tokens 以下** であること
- 上記を満たせば月額試算は $6.79 以内に収まり、$20 上限に対して十分なマージンあり

## No.3 — BALLDONTLIE API キーの確認

### 目的
C-05（スケジュール取得機能）で使用する BALLDONTLIE API キーが有効であることを確認する。

### 手順

1. ブラウザで以下にアクセス:  
   `https://app.balldontlie.io`

2. アカウントにログインする

3. ダッシュボードで以下を確認:
   - **API キー** が発行・表示されていること
   - プランが **無料ティア（Free）** であること
   - APIキーの有効期限・ステータスが **Active** であること

4. API キーの文字列をコピーして手元に保管する（No.5 で使用）

### 確認基準
- API キーが存在し、Active であること
- 無料ティアの制限（1,000リクエスト/日）が requirements_p2_v0.12.md の想定内であること

### 備考
- BALLDONTLIE は4時間ごとに1回呼び出し = 6回/日（制限の 0.6%）
- API キーを紛失した場合はダッシュボードから再発行可能

## No.4 — Render アカウント・プランの確認

### 目的
フェーズ2のホスティング環境（Render Starter + Persistent Disk）の設定方法を把握する。

### 手順

#### 4-1. アカウントの確認

1. ブラウザで以下にアクセス:  
   `https://render.com`

2. アカウントが存在しない場合: **Sign Up** からアカウントを作成する
   - GitHubアカウントで登録推奨（デプロイ連携が容易になる）

3. アカウントにログインする

#### 4-2. Starter プランの確認

1. ダッシュボード → **Billing** を開く

2. 現在のプランを確認する:
   - **Free プラン**: Starter プランへのアップグレードが必要
   - **Starter プラン以上**: そのまま利用可能

3. アップグレードが必要な場合:
   - **Upgrade Plan** → **Starter** を選択（$7/月）
   - クレジットカードを登録・支払い設定を完了する

#### 4-3. Persistent Disk の設定方法を把握する

製造後のデプロイ時に必要な手順（今は把握のみ・実施不要）:

1. サービス作成時またはサービス設定画面で **Disks** セクションを開く

2. **Add Disk** をクリックし以下を設定:
   - **Name:** `nba-news-db`
   - **Mount Path:** `/data`
   - **Size:** `1 GB`
   - 月額: **$0.25**

3. アプリの環境変数に以下を設定（製造後に実施）:
   ```
   DATABASE_URL=/data/nba_news.db
   ```

### 確認基準
- Render アカウントが存在し、ログインできること
- Starter プラン以上であること（または Starter へのアップグレード手順を把握していること）
- Persistent Disk の追加方法を把握していること

### 月額コスト確認

| 項目 | 費用 |
|------|------|
| Render Starter（Web Service） | $7.00/月 |
| Render Persistent Disk（1GB） | $0.25/月 |
| Claude API（推定） | $3.40/月 |
| **合計** | **$10.65/月** |

## No.5 — `.env` ファイルの準備

### 目的
ローカル開発・テスト用の `.env` ファイルを作成する。製造後に同じ値を Render 環境変数にも設定する。

### 手順

#### 5-1. SECRET_KEY の生成

ターミナルで以下のコマンドを実行し、SECRET_KEY を生成する:

```bash
openssl rand -hex 32
```

出力例（実際の値は毎回異なる）:
```
a3f8c2e1d9b7456f0e2a8c4b6d1e3f5a7c9e2b4d6f8a0c2e4b6d8f0a2c4e6b8
```

生成された文字列を手元に保管する。

#### 5-2. `.env` ファイルの作成

プロジェクトルート（`app/` ディレクトリ）に `.env` ファイルを作成し、以下の内容を記載する:

```env
# Claude API
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxx

# BALLDONTLIE API
BALLDONTLIE_API_KEY=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

# JWT認証
SECRET_KEY=<openssl rand -hex 32 で生成した値>

# 初期ユーザー
INIT_USERNAME=<ログインユーザー名>
INIT_USER_PASSWORD=<ログインパスワード>
```

各変数を実際の値に置き換えること:

| 変数 | 値の取得元 |
|------|-----------|
| `ANTHROPIC_API_KEY` | `https://console.anthropic.com` → API Keys |
| `BALLDONTLIE_API_KEY` | No.3 の手順で取得した値 |
| `SECRET_KEY` | 5-1 で生成した値 |
| `INIT_USERNAME` | 任意（家族が使うログイン名） |
| `INIT_USER_PASSWORD` | 任意（十分な強度のパスワード） |

#### 5-3. `.gitignore` の確認

`.env` がバージョン管理に含まれないことを確認する:

```bash
cat .gitignore | grep ".env"
```

`.env` の記載がない場合は追記する:

```bash
echo ".env" >> .gitignore
```

### 確認基準
- `.env` ファイルが作成され、全5変数が入力されていること
- `.gitignore` に `.env` が記載されていること

### 重要注意事項

> ⚠️ **SECRET_KEY はソースコードにハードコードしないこと。**  
> ローカルは `.env` ファイル、Render 本番環境は Render の **Environment Variables** に設定すること。  
> （詳細設計書 §2-2 参照）

> ⚠️ **`.env` ファイルを Git にコミットしないこと。**  
> GitHub に流出した場合、JWT 認証が無効化される。

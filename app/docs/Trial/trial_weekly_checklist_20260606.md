- **対象**: NBAニュース翻訳アプリ（個人利用）

## 事前準備

```powershell

cd backend
python main.py

# フロントエンド起動（起動していない場合）

npm run dev
```

ブラウザで `http://localhost:5173` を開く。

|---|---|---|---|
| A-1 | `title_ja`（見出し）が自然な日本語か | 意味が通じる日本語であること | OK |
| A-2 | `summary_ja`（要約）が自然な日本語か | 読了に支障がないレベル | OK |

## 本運用移行判断基準（2〜4週間後）

以下をすべて満たした場合に手順11（本運用）へ移行する。

| 基準項目 | 合格条件 |
|---|---|
| コスト | 月次コストが試算（$5）の150%以内（$7.50以下） |
| 翻訳品質 | 読了に支障がないレベル（A-1〜A-3が継続して合格） |
| カテゴリ分類精度 | 目視確認で80%以上の正確性（B-1〜B-4が継続して合格） |
| バッチ安定性 | トライアル期間中のバッチ成功率95%以上（E-1が継続して合格） |

以下のいずれか1つに該当した場合は中止・再設計を検討する。

| 基準項目 | 中止条件 |
|---|---|
| コスト超過 | 月次コストが試算の200%（$10.08）を超えた場合 |
| RSSソース不安定 | フェールオーバーが週3回以上発生する場合 |
| 翻訳品質不足 | 意味の取れない翻訳が全体の30%を超える場合 |

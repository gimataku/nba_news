## 本運用移行判断基準（project_plan_p2_v1.15.md 手順10より）

以下をすべて満たした場合に手順11（ドキュメントベースライン設定）→手順12（本運用）へ移行する。

| 基準項目 | 合格条件 |
|---|---|
| コスト | 月次コストが試算の150%以内 |
| 翻訳品質 | 読了に支障がないレベル（新カテゴリ含む） |
| カテゴリ分類精度 | 目視確認で80%以上の正確性 |
| バッチ安定性 | トライアル期間中のバッチ成功率95%以上 |
| 認証機能 | 想定外のアクセスがないこと |

以下のいずれか1つに該当した場合は中止・再設計を検討する。

| 基準項目 | 中止条件 |
|---|---|
| コスト超過 | 月次コストが試算の200%を超えた場合 |
| 認証迂回 | 想定外の第三者アクセスが確認された場合 |
| 翻訳品質不足 | 意味の取れない翻訳が全体の30%を超える場合 |
| インフラ不安定 | バッチ・サーバーの成功率が95%を下回った場合 |

> **注記：** C項目（ネタバレ判定）はオフシーズンのため、シーズン開幕（2026年10月予定）までは「本運用移行判断基準」の評価対象から除外する。シーズン開幕後、最低2週分のC項目確認結果が出るまでは本運用への正式移行を見送ること（Takumi様2026-06-28承認）。

[YEL-01] TC-P2-02-5 の入力値・前提条件が不明確
該当箇所：§2 T-P2-02 ケース TC-P2-02-5

  現在の記述：
    | TC-P2-02-5 | `POST /api/auth/login` に認証なしでアクセス | HTTP 200（認証不要エンドポイント） |

  問題：「認証なし」の意味が曖昧。
  - 「JWT トークンなし」なら、ボディに正しいユーザー名・パスワードを含めれば HTTP 200
  - 「ユーザー名・パスワードも含めない」なら、バリデーションエラーで HTTP 422 または 400

  本ケースの目的が「JWT 不要のエンドポイントであること」の確認なら、
  「JWT ヘッダーなし・正しいユーザー名・パスワードをボディに含む」と明記すべき。

  修正案：
    | TC-P2-02-5 | JWT ヘッダーなし・正しいユーザー名&パスワードで POST /api/auth/login |
                   HTTP 200（JWT 認証不要エンドポイント） |

[YEL-02] シナリオ7 にログアウトシナリオが含まれていない
該当箇所：§4 総合テスト シナリオ7「ログイン・ログアウトフロー」

  現在の記述はステップ1（ログイン画面表示）・ステップ2（ログイン成功）・
  ステップ3（誤パスワードでログイン失敗）の3ステップで終わっている。

  シナリオタイトルに「ログアウト」が含まれているにも関わらず、
  ログアウト操作とその後の画面遷移が未記載。

  追加が必要なステップ：
    ステップ4: ログアウトボタンをクリック → ログイン画面に戻ること
    （オプション）ステップ5: ページリロード後の挙動を確認
    （メモリ保持の場合はログアウト状態・D-06 の確定内容に依存する）

[YEL-03] カバレッジ 80% の対象範囲・計測方法が未定義
該当箇所：§1-1 テスト方針 フェーズ1との差分表

  「カバレッジ目標 80%以上（pytest --cov）」と記載されているが、以下が未定義。

  (a) 対象スコープ：バックエンドのみか、フロントエンドも含むか。
      フロントエンド（React）のカバレッジは pytest ではなく vitest --coverage 等が必要。
  (b) 除外対象：config.py・main.py の startup ロジック等、テストしづらいコードの
      除外設定（pytest-cov の --omit オプション）を明示すべき。
  (c) 計測単位：行カバレッジ（line）かブランチカバレッジ（branch）か。

  修正方針：§1-1 に以下を追記すること。
    - カバレッジ対象：バックエンド（Python / pytest-cov）
    - 除外対象ファイル例：config.py・main.py（startup）・migrations
    - 計測単位：行カバレッジ（line coverage）
    - フロントエンドカバレッジ：今フェーズは対象外（将来課題）

[YEL-04] ブルートフォース・連続ログイン失敗のテストケースがない
該当箇所：§2 T-P2-01 / §4 シナリオ7
セキュリティ観点

  T-P2-01 のテストケースは正常・誤パスワード・存在しないユーザーの 3 ケース。
  連続ログイン失敗時の挙動（レート制限・アカウントロック等）のテストが存在しない。

  詳細設計書 YEL-05 のブルートフォース対策が未確定のため、テストも書けていない状態。
  対応方針：
    - 詳細設計書 YEL-05 でブルートフォース対策の方針を確定後、
      本テスト設計書に対応するテストケースを追加すること
    - 方針を「対策なし（リスク受け入れ）」とした場合でも、
      その旨をテスト設計書に「対象外：ブルートフォース対策なし（design decision）」
      と明記することを推奨する

[GRN-01] 詳細設計書 §7 との HTTPステータスコード表現の不統一
該当箇所：§2 TC-P2-02-1

  TC-P2-02-1（Authorization ヘッダーなし）の期待結果は「HTTP 403 または 401」と
  正しく記述されており、FastAPI の HTTPBearer の実際の挙動を反映している。

  詳細設計書 §7 には Authorization ヘッダーなしのケースが未記載（→ 詳細設計 YEL-03 で指摘済み）。
  詳細設計書が修正されたら、テスト設計書の期待結果との表現を合わせること。
  テスト設計書の記述自体は正しい。

## 1. プロジェクト概要

海外NBAニュースサイト（Hoops Rumors等）のRSSフィードを定時取得し、Claude APIで日本語要約・カテゴリ分類して表示するWebアプリ（ローカルサーバー）を開発する。

| 項目 | 内容 |
|---|---|
| 目的 | 海外NBAニュースを日本語で快適に読む |
| 利用者 | 個人（非公開） |
| 利用形態 | Webブラウザ（ローカルサーバー） |
| 参照ドキュメント | 要件定義書 v0.6（`requirements_v0.6.md`） |

---

- [x] ESPN RSS フィードの利用規約確認 → 「改変禁止」条項により除外

- [x] チームフィルタに必要なメタ情報（チーム名タグ等）がRSSに含まれるか確認 → Hoops Rumors・Hoops Wire・The Cold Wire NBAいずれも`<category>`タグにチーム名が含まれることを実機確認済み
- [x] ESPN RSSが利用不可だった場合の代替ニュースソースの特定 → Hoops Wire（FO1）・The Cold Wire NBA（FO2）に確定
- [x] ローカル環境でのバッチスケジューリング方式の確認 → 前回取得から4時間経過で実行（cron不使用）
- [x] Claude API レートリミットと記事処理数の整合性確認 → 問題なし（60リクエスト/分に対し1秒/件インターバルで対応）

- **期間**: 2〜4週間
- **確認項目**:
  - [ ] Claude APIの実際のコスト実績（試算との乖離確認）
  - [ ] 翻訳・要約の品質確認

  - [ ] ネタバレ判定の検出漏れ確認
  - [ ] 定時バッチの安定動作確認

#### 本運用移行基準（すべてを満たすこと）

| 基準項目 | 合格条件 |
|---|---|
| コスト | 月次コストが試算の150%以内 |
| 翻訳品質 | 読了に支障がないレベル（意味が通じる日本語であること） |
| カテゴリ分類精度 | 目視確認で80%以上の正確性 |
| バッチ安定性 | トライアル期間中のバッチ成功率95%以上 |

#### 中止・再設計基準（いずれか1つに該当した場合）

| 基準項目 | 中止条件 |
|---|---|
| コスト超過 | 月次コストが試算の200%を超えた場合 |
| RSSソース不安定 | 主要RSSソースの過半が安定取得できない場合 |
| 翻訳品質不足 | 意味の取れない翻訳が全体の30%を超える場合 |

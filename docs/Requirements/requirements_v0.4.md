- **ステータス**: Draft

---

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| 目的 | 海外NBAニュースを日本語で快適に読む |
| 利用者 | 個人（非公開） |
| 利用形態 | Webブラウザ（ローカルサーバー） |
| 更新頻度 | 1日2回（朝・夕の定時バッチ） |
| お気に入りチーム | San Antonio Spurs（1チーム） |
| 将来拡張 | NFL対応（優先度低・フェーズ2） |

---

## 2. 機能要件

| ID | 機能名 | 内容 | 優先度 |
|---|---|---|---|
| F-01 | ニュース取得 | 公式RSS / 認可API経由でNBAニュースを定時取得（1日2回） | 高 |
| F-02 | 日本語翻訳・要約 | 見出し＋本文をClaude APIで要約・日本語化（300〜500字程度） | 高 |
| F-03 | カテゴリ自動分類 | トレード / 契約 / 試合結果 / コラム の4分類をClaude APIで自動判定 | 高 |
| F-04 | お気に入りチームフィルタ | 設定した1チーム（Spurs）に関連するニュースのみ表示する切替機能 | 高 |
| F-05 | ネタバレ防止 | 試合結果カテゴリの詳細スコア・スタッツをデフォルト非表示（クリックで解除） | 高 |
| F-06 | 元記事リンク | 全文は元サイトへのリンクで参照（著作権対応） | 高 |
| F-07 | API使用上限管理 | Claude APIの月次コスト上限を設定、超過時は翻訳スキップ＋アラート表示 | 中 |
| F-08 | NFL拡張対応 | スポーツ種別を切り替えられる構造にしておく（フェーズ2実装） | 低 |

---

### 表示仕様（F-02補足）

- 見出し（翻訳済み）
- 要約本文（Claude APIが独自に生成・日本語化、300〜500字程度）
- 元記事へのリンク・ソースクレジット
- 記事の全文翻訳・全文保存は行わない（著作権対応）

---

### カテゴリ定義と境界条件（F-03補足）

| カテゴリ | 定義 | 境界条件 |
|---|---|---|
| トレード | 移籍・トレード噂・成立報道 | 交渉段階も含む |
| 契約 | サイン・契約延長・FA動向 | 契約金額の報道も含む |
| 試合結果 | 試合当日〜翌日の速報・ボックススコア記事 | スコア・スタッツを主要情報として含む記事 |
| コラム | 試合分析・選手評価・戦術考察・インタビュー等 | スコアが引用として含まれる場合も「コラム」に分類 |

**ネタバレ防止のカテゴリ適用方針：**
「試合結果」カテゴリはネタバレ防止を適用する。「コラム」カテゴリでも本文中にスコア・スタッツが含まれると判定された場合は、同様にネタバレ防止を適用する（安全側の設計）。

---

### ネタバレ防止仕様（F-05補足）

- **対象カテゴリ**：「試合結果」、および本文にスコア・スタッツが含まれると判定された「コラム」
- **ネタバレ検出対象**：
  - RSSニュース本文中のスコア記述（テキスト）
  - BALLDONTLIE APIから取得したボックススコア・スタッツデータ
- **要約生成時の制御**：「試合結果」カテゴリと判定された場合、Claude APIはスコア・スタッツを要約文に含めない（プロンプトで制御）
- **スコア・スタッツの表示データソース**：BALLDONTLIE API（RSSの本文スコアは表示に使用しない）
- **非表示対象**：勝敗・スコア・ボックススコア・個人スタッツ
- **デフォルト**：非表示（ぼかし表示）
- **解除**：「スコア・スタッツを表示」ボタンをクリックで解除
- **全体設定**：アプリ全体でネタバレ防止ON/OFFを切替可能

---

### API使用上限管理仕様（F-07補足）

| 項目 | 内容 |
|---|---|
| 月次上限額 | $20（Anthropic Console上で設定） |
| 超過時の動作 | 翻訳・要約をスキップし、見出し（原文のまま）のみ表示 |
| アラート表示位置 | ヘッダー部に「今月のAPI上限に達しました」バナー表示 |
| 月次リセット | 毎月1日（Claude APIの請求サイクルに合わせる） |

## 3. 非機能要件

| 項目 | 内容 |
|---|---|
| 法的制約 | スクレイピング禁止。RSSまたは公式APIのみ使用。記事全文の翻訳・保存不可 |
| コスト制約 | Claude APIは月上限$20を設定。無料RSS優先、有料APIは最小化 |
| データ保持 | 取得済みニュースはローカルDB（SQLite）に保存。記事取得日から30日分を保持し、定時バッチ実行時に30日超データを自動削除する |
| セキュリティ | APIキーは環境変数で管理。ローカルのみアクセス可 |
| パフォーマンス | ページ初期表示は1秒以内（キャッシュ済みデータ表示） |

## 5. 主要リスクサマリー（risk_report_v0.2.md 参照）

### 法的リスク
- ニュースソースはHoops Rumors（主軸）・RealGM（補助）・HoopsHype（補助）を採用（ESPN・NBA.comはニュース取得に使用しない）
- 翻訳・要約はClaude APIが独自生成。RSSコンテンツの直接翻訳・転載は行わない
- 個人利用・非公開・元記事リンク表示・ソースクレジット表示を厳守

### コストリスク
- Claude API：月次上限$20をConsole上で設定（推定月額$5）
- ニュースRSS・BALLDONTLIE APIはすべて無料ティアで対応

### 技術リスク
- RSSフィード提供停止・仕様変更の可能性（代替ソースを複数保持することで対応）
- カテゴリ自動分類の精度（Claude APIのプロンプト設計で対応）
- ネタバレ判定の精度（スコア・スタッツ含む「コラム」への安全側適用で補完）
- Hoops Rumors等のRSSに含まれるチームメタ情報が未確認（基本設計着手前に実機確認が必要）

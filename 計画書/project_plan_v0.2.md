## 1. プロジェクト概要

海外NBAニュースサイト（ESPN等）のRSSフィードを定時取得し、Claude APIで日本語要約・カテゴリ分類して表示するWebアプリ（ローカルサーバー）を開発する。

| 項目 | 内容 |
|---|---|
| 目的 | 海外NBAニュースを日本語で快適に読む |
| 利用者 | 個人（非公開） |
| 利用形態 | Webブラウザ（ローカルサーバー） |
| 参照ドキュメント | 要件定義書 v0.3（`requirements_v0.3.md`） |

---

### 手順2：リスク事前調査 ✅

- **アウトプット**: リスク調査レポート（簡易版）
- **成果物**: `risk_report_v0.1.md`（作成予定）
- **目的**: 設計着手前に法的・金銭的・技術的リスクの致命的な問題がないことを確認する

#### 法的調査
- [ ] ESPN RSS フィードの利用規約確認（個人利用の定義、商用・非商用の区別）
- [ ] NBA.com RSS フィードの利用規約確認
- [ ] その他利用予定RSSソースの利用規約確認
- [ ] 日本著作権法における「要約表示」の扱いの確認
- [ ] 翻訳要約の表示が引用の範囲内かどうかの確認

#### コスト調査
- [ ] Claude API（claude-sonnet-4系）の料金体系確認
- [ ] 1日2回取得 × 想定記事数（20〜30件/回）× 要約トークン数での月次コスト試算
- [ ] Claude API の月次上限設定方法の確認
- [ ] 無料RSSで賄えるニュースソースの範囲確認

#### 技術調査
- [ ] ESPN・NBA.com のRSSフィード提供状況の確認
- [ ] RSSフィードに含まれる情報量（本文量）の確認
- [ ] チームフィルタに必要なメタ情報（チーム名タグ等）がRSSに含まれるか確認
- [ ] ESPN RSSが利用不可だった場合の代替ニュースソースの特定
- [ ] ローカル環境でのバッチスケジューリング方式の確認（cron / タスクスケジューラ / 常駐プロセス。PCスリープ時の挙動含む）
- [ ] Claude API レートリミットと記事処理数の整合性確認

### 手順10：トライアル運用 ⬜

- **期間**: 2〜4週間
- **確認項目**:
  - [ ] Claude APIの実際のコスト実績（試算との乖離確認）
  - [ ] 翻訳・要約の品質確認
  - [ ] カテゴリ分類の精度確認
  - [ ] ネタバレ判定の検出漏れ確認
  - [ ] 定時バッチの安定動作確認

#### 本運用移行基準（すべてを満たすこと）

| 基準項目 | 合格条件 |
|---|---|
| コスト | 月次コストが試算の150%以内 |
| 翻訳品質 | 読了に支障がないレベル（意味が通じる日本語であること） |
| カテゴリ分類精度 | 目視確認で80%以上の正確性 |
| バッチ安定性 | トライアル期間中のバッチ成功率95%以上 |

#### 中止・再設計基準（いずれか1つに該当した場合）

| 基準項目 | 中止条件 |
|---|---|
| コスト超過 | 月次コストが試算の200%を超えた場合 |
| RSSソース不安定 | 主要RSSソースの過半が安定取得できない場合 |
| 翻訳品質不足 | 意味の取れない翻訳が全体の30%を超える場合 |

| 区分 | 内容 |
|---|---|
| 法的制約 | スクレイピング禁止。RSSまたは公式APIのみ使用。記事全文の翻訳・保存不可 |
| コスト制約 | Claude APIは月次上限を設定。無料RSS優先、有料APIは最小化 |

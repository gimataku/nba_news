## 1. 法的リスクの最終評価

### 1-1. ニュースソース利用規約

| ソース | 利用規約状況 | 採用判定 | 対処内容 |
|---|---|---|---|
| **Hoops Rumors**（主軸） | 明示的な禁止条項なし（独立ブログ形式） | ✅ 採用 | 元記事リンク・クレジット表示を実装 |
| **Hoops Wire**（FO1） | 明示的な禁止条項なし | ✅ 採用 | 同上 |
| **The Cold Wire NBA**（FO2） | 明示的な禁止条項なし | ✅ 採用 | 同上 |
| ESPN | 「改変禁止」条項あり | ❌ 除外済み | 設計から完全除外 |
| NBA.com | 「抜粋・編集禁止」条項あり | ❌ 除外済み（ニュース用途） | 設計から完全除外 |
| HoopsHype / RealGM | RSS 404 | ❌ 除外済み | 物理的に利用不可 |

**設計上の対処（全ソース共通）：**
- RSSコンテンツを直接翻訳・転載しない。Claude APIが**独自に要約を生成**する
- 全記事に**元記事URLとソースクレジット**を表示（`link`カラムを`source_url`としてAPIレスポンスで返す・`source`カラム、`NewsCard.jsx`で実装済み）
- ローカル環境のみで動作。外部公開なし

### 1-2. 日本著作権法

**評価：個人利用・ローカル限定の条件下で適法の範囲内。**

| 根拠 | 内容 |
|---|---|
| 著作権法第47条の6 | 個人的・家庭内等の閉鎖的範囲内での利用であれば翻訳・翻案も許容 |
| 「血液型と性格」要約引用事件（東京地判平成10年10月30日） | やむを得ない範囲での要約引用は許容。要約は全文複製より著作権者の利益を損なわない |

**遵守事項（設計で対処済み）：**
- ローカルのみ（`host="127.0.0.1"`でFastAPI起動）
- 元記事リンクを全記事に表示
- 全文保存・全文翻訳は行わない（`summary_ja`は要約のみ）
- 要約はClaude APIが独自生成（RSSコンテンツの転載ではない）

**残存リスク：** AI生成翻訳要約の著作権上の扱いは法整備途上。ただし個人利用・ローカル限定・要約独自生成の条件下では、現時点の法解釈上リスクは低い。受容する。

### 1-3. 法的リスク総合判定

✅ **許容範囲内。製造進行に支障なし。**

## 2. コストリスクの最終評価

### 2-1. Claude API コスト

## 3. 技術リスクの最終評価

### 3-1. RSSソース安定性

| リスク | 内容 | 対処状況 |
|---|---|---|
| Hoops Rumors停止 | 主軸ソースの停止 | ✅ Hoops Wire（FO1）→The Cold Wire NBA（FO2）の2段階フェールオーバー設計済み |
| 全ソース同時停止 | 3ソースすべてが停止 | ⚠️ 残存リスク（受容）。前回取得データをそのまま表示し、バナーで通知 |
| RSS仕様変更 | フィードのXML構造が変わる | ⚠️ 残存リスク（受容）。feedparserは一般的なRSS/Atom形式に対応 |

### 3-2. Claude API

| リスク | 内容 | 対処状況 |
|---|---|---|
| レートリミット超過 | 1秒/件インターバルで対応 | ✅ 設計済み |
| 429（月次上限）到達 | 翻訳スキップ＋英語見出し保存 | ✅ 設計済み |
| JSONパースエラー・バリデーション失敗 | Claude Client | `title_ja=title_original`・`summary_ja=NULL`・`category=NULL`・`has_score=False`で保存・バッチ継続 | ✅ 設計済み |
| モデル廃止・変更 | `config.py`の`CLAUDE_MODEL`を変更するのみ | ✅ 対応容易 |
| 料金改定 | 上限$20が不足する可能性 | ⚠️ 残存リスク（受容）。製造着手前に最新料金を再確認 |

### 3-3. BALLDONTLIE API

| リスク | 内容 | 対処状況 |
|---|---|---|
| 無料ティア廃止・有料化 | スコア表示機能が使えなくなる | ⚠️ 残存リスク（受容）。`score_data=NULL`・ボタン非表示で機能劣化にとどまる |
| スコアデータ取得失敗 | `score_data=NULL`で保存 | ✅ 設計済み |
| SpursチームID変更 | `config.py`の`SPURS_TEAM_ID`を変更するのみ | ✅ 対応容易 |

### 3-4. ローカル環境依存

| リスク | 内容 | 対処状況 |
|---|---|---|
| PCスリープ問題 | バッチが定刻に実行されない | ✅ 経過時間チェック方式で設計済み（4時間経過で実行） |
| PC買い替え・OS変更 | 環境再構築が必要 | ✅ `setup.md`（環境構築手順書）を成果物として定義済み |
| SQLiteファイル破損 | データ消失 | ⚠️ 残存リスク（受容）。個人利用のため定期バックアップは手動対応 |

### 3-5. 技術リスク総合判定

✅ **許容範囲内。主要リスクはすべて設計で対処済み。残存リスクは軽微で受容可能。**
